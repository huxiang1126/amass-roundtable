# 集团圆桌会议 · Claude Code Skill v1.1

三位集团高管（HR总监 · 总顾问 · COO）围绕你的问题展开多视角讨论、博弈与决策。

**v1.1 新特性：Agent Teams 驱动** — 每位高管运行在独立上下文中，真正的多角色独立思考，不共享记忆，避免观点趋同。

支持跨会话记忆、会议纪要自动归档、行动项追踪、可视化图表输出、动态角色扩展。

---

## 安装

### 前提条件

- 已安装 [Claude Code](https://claude.ai/download)（桌面应用或终端版均可）
- 已安装 Git（Mac 自带）

### 第一步：下载

打开「终端」，粘贴：

```bash
git clone https://github.com/huxiang1126/amass-roundtable.git
```

### 第二步：安装到你想要的位置

```bash
bash amass-roundtable/setup.sh /你想安装的路径
```

路径可以是任意位置，文件夹不存在会自动创建：

```bash
# 示例
bash amass-roundtable/setup.sh ~/Desktop/圆桌会议
bash amass-roundtable/setup.sh ~/Documents/我的圆桌
bash amass-roundtable/setup.sh /你已有的项目目录
```

安装脚本会自动：
- 复制 Skill 文件到 `.claude/skills/`
- 开启 Agent Teams 功能（写入 `.claude/settings.json`）
- 创建数据目录和角色库
- 配置 `.gitignore` 保护敏感信息

看到 `安装完成！` 就装好了。

### 第三步：打开 Claude Code

1. 打开 **Claude Code**
2. 选择第二步中安装到的那个文件夹作为项目目录
3. 输入 `/roundtable`，出现命令提示说明安装成功

> **重要**：Claude Code 只识别当前项目目录下的 Skill。确保你选的是安装了 Skill 的文件夹。

### 安装后的目录结构

```
你选择的文件夹/
├── .claude/
│   ├── settings.json             ← Agent Teams 配置
│   └── skills/
│       └── roundtable/
│           └── SKILL.md          ← Skill 核心文件
└── roundtable/
    ├── roles/                    ← 角色库（3核心 + 5扩展）
    │   ├── hr-director.md        ← HR总监
    │   ├── strategic-advisor.md  ← 总顾问
    │   ├── coo.md                ← COO
    │   ├── cfo.md                ← 财务总监
    │   ├── brand-director.md     ← 品牌总监
    │   ├── supply-chain.md       ← 供应链总监
    │   ├── sales-director.md     ← 销售总监
    │   └── legal-counsel.md      ← 法务总监
    ├── minutes/                  ← 会议纪要（自动生成）
    └── .gitignore
```

## 使用方法

### 启动会议

```
/roundtable 门店店长绩效怎么考核
```

或不带议题：

```
/roundtable
```

### 首次使用

首次启动会进入**公司信息对齐**。系统会向你提问公司基本盘、组织架构、经营现状和核心痛点。

对齐完成后信息保存到 `roundtable/company-profile.md`，后续会议自动加载。

> 对齐过程中直接打字回答，不需要加 `/roundtable` 前缀。

### 会议中指令

| 你说 | 效果 |
|------|------|
| 直接说话 | 抛议题或补充信息，高管们继续讨论 |
| `继续` | 讨论继续深入 |
| `追问 HR总监` | 指定某角色深入展开 |
| `拍板` | 结束议题，记录决议 |
| `拉 财务总监` | 引入新角色（没有的自动创建） |
| `踢 财务总监` | 移出扩展角色（核心三人不可踢） |
| `画` | 生成可视化图表 |
| `散会` | 保存纪要 + 全景图，更新行动追踪 |
| `回顾` | 查看历史纪要和未完成行动项 |
| `更新档案` | 更新公司档案 |

### 议题示例

```
"最近3个店长都辞职了，怎么回事？"
"我想把华东区独立成事业部，可行吗？"
"门店业绩下滑但店长说是天气原因，我不太信"
"明年预算砍30%，哪些部门优先保？"
```

## 核心角色

| 角色 | 视角 | 独立上下文 |
|------|------|-----------|
| HR总监 | 看人、看制度、看组织病灶 | 独立 Teammate |
| 总顾问 | 看系统、看结构、看战略失真 | 独立 Teammate |
| COO | 看落地、看堵点、看运行损耗 | 独立 Teammate |

v1.1 中每位高管运行在**独立的 Agent Teammate** 中，各自读取公司档案、各自形成判断，不共享思考过程，确保观点真正独立。

## 扩展角色

### 预装角色

| 角色 | 触发词 |
|------|--------|
| 财务总监 | `拉 财务总监` / `拉 CFO` |
| 品牌总监 | `拉 品牌总监` |
| 供应链总监 | `拉 供应链总监` |
| 销售总监 | `拉 销售总监` |
| 法务总监 | `拉 法务总监` |

### 动态创建

直接说 `拉 IT总监` 或 `拉 电商运营总监`，系统自动创建完整角色定义并保存，下次会议直接加载。

## 讨论机制

| 轮次 | 内容 | 自动输出 |
|------|------|----------|
| 第1轮 · 独立研判 | 各角色独立分析，标注分歧 | 矛盾结构图 |
| 第2轮 · 交叉质询 | 互相质疑、挑战 | — |
| 第3轮 · 收敛决议 | 共识 + 行动项 | 决策路径图 |

散会时生成「会议全景图」。讨论根据实际深度自然推进，不是死板模板。

## 记忆系统

| 文件 | 说明 |
|------|------|
| `roundtable/company-profile.md` | 公司档案（首次对齐后生成） |
| `roundtable/action-tracker.md` | 行动追踪（跨会议累积） |
| `roundtable/minutes/日期-主题.md` | 会议纪要（每次散会保存） |
| `roundtable/roles/*.md` | 角色定义（预装 + 动态创建） |

每次 `/roundtable` 自动加载档案 + 最近3份纪要 + 行动追踪。

---

## 注意事项

- 公司档案和纪要含敏感信息，`.gitignore` 已排除
- Agent Teams 是实验性功能，安装脚本已自动开启
- 建议定期用「回顾」检查未完成行动项
- 公司发生重大变化时用「更新档案」同步
