# 集团圆桌会议 · Claude Code Skill

三位集团高管（HR总监 · 总顾问 · COO）围绕你的问题展开多视角讨论、博弈与决策。

支持跨会话记忆、会议纪要自动归档、行动项追踪、可视化图表输出、动态角色扩展。

---

## 安装

### 第一步：下载

打开「终端」应用（Mac 在「启动台」→「其他」→「终端」），粘贴以下命令：

```bash
cd ~/Desktop
git clone https://github.com/huxiang1126/amass-roundtable.git
```

桌面会出现一个 `amass-roundtable` 文件夹。

### 第二步：安装到你的项目

假设你想在桌面建一个叫 `圆桌会议` 的文件夹来使用，在终端继续输入：

```bash
mkdir -p ~/Desktop/圆桌会议
cd ~/Desktop/amass-roundtable
bash setup.sh ~/Desktop/圆桌会议
```

看到 `安装完成！` 就装好了。

### 第三步：在 Claude Code 桌面应用中打开

1. 打开 **Claude Code 桌面应用**
2. 点击左上角的项目名称（或新建对话时选择项目目录）
3. 选择刚才创建的 `圆桌会议` 文件夹
4. 在对话框输入 `/roundtable` ，如果出现命令提示，说明安装成功

> **重要**：必须选择安装了 Skill 的那个文件夹作为项目目录，否则 Claude Code 找不到 `/roundtable` 命令。

## 使用方法

### 启动会议

在 Claude Code 对话框中输入：

```
/roundtable 门店店长绩效怎么考核
```

或者不带议题启动：

```
/roundtable
```

### 首次使用

首次启动会进入**公司信息对齐**流程。三位高管会轮流向你提问，了解公司基本盘、组织架构、经营现状和核心痛点。

对齐完成后，信息会保存到 `roundtable/company-profile.md`，后续会议自动加载，不需要重复介绍。

> **提示**：对齐过程中直接打字回答即可，不需要每次都加 `/roundtable` 前缀。只有第一次启动会议时需要输入 `/roundtable`。

### 会议中指令

| 指令 | 说明 |
|------|------|
| `继续` | 让讨论继续深入 |
| `追问 HR总监` | 指定某个角色深入展开（也可追问 总顾问 / COO） |
| `拍板` | 结束当前议题，记录决议 |
| `拉 财务总监` | 引入角色：角色库有则加载，没有则自动创建 |
| `踢 财务总监` | 将扩展角色移出讨论（核心三人不可踢） |
| `画` | 根据当前讨论内容，生成可视化图表（矛盾结构图/决策路径/组织关系图等） |
| `散会` | 保存会议纪要 + 全景图，更新行动追踪表 |
| `回顾` | 查看历史纪要和未完成的行动项 |
| `更新档案` | 根据讨论内容更新公司档案 |

### 抛议题

正常说话就行，不需要特殊格式：

```
"最近3个店长都辞职了，怎么回事？"
"我想把华东区独立成事业部，可行吗？"
"门店业绩下滑但店长说是天气原因，我不太信"
"明年预算砍30%，哪些部门优先保？"
```

## 核心角色

| 角色 | 视角 |
|------|------|
| HR总监 | 看人、看制度、看组织病灶 |
| 总顾问 | 看系统、看结构、看战略失真 |
| COO | 看落地、看堵点、看运行损耗 |

## 扩展角色

### 预设角色（开箱即用）

| 角色 | 触发词 | 视角 |
|------|--------|------|
| 财务总监 | `拉 财务总监` / `拉 CFO` | 投入产出、现金流、成本结构 |
| 品牌总监 | `拉 品牌总监` | 品牌认知、客户感知、触点一致性 |
| 供应链总监 | `拉 供应链总监` / `拉 供应链` | 库存、采购、物流、补货 |
| 销售总监 | `拉 销售总监` | 终端成交、门店执行、区域差异 |
| 法务总监 | `拉 法务总监` / `拉 法务` | 合规、劳动关系、合同、风险预警 |

### 不在库里的角色？直接拉

不需要提前创建。直接说 `拉 IT总监` 或 `拉 客服总监`，系统会自动按完整模板创建角色（角色定位 + 底层锚点 + 工作方法 + 职责范围 + 发言风格 + 典型质疑），创建后自动保存到角色库，下次会议直接加载。

讨论中角色也会主动建议"这个问题需要XX视角"，主持人确认后自动拉入。

## 可视化输出

讨论过程中会自动生成可视化图表（Mermaid 语法），帮你更直观地理解讨论内容：

- **矛盾结构图** — 第一轮独立研判后自动生成，展示各方判断差异
- **决策路径图** — 收敛后自动生成，展示从现状到目标的行动路径
- **组织关系图** — 讨论架构问题时生成
- **对比矩阵** — 多方案比较时生成
- **时间线图** — 涉及阶段性推进时生成
- **会议全景图** — 散会时自动生成，汇总核心判断与行动项

也可以随时说「画」手动触发。

## 记忆系统

```
roundtable/
├── company-profile.md     ← 公司档案（首次对齐后自动生成）
├── action-tracker.md      ← 行动追踪表（自动维护）
├── roles/                 ← 可拉入的扩展角色
│   ├── cfo.md
│   ├── brand-director.md
│   ├── supply-chain.md
│   ├── sales-director.md
│   └── legal-counsel.md
└── minutes/               ← 会议纪要（散会时自动生成）
    ├── 2026-03-25-门店绩效.md
    └── 2026-03-26-区域架构.md
```

- **公司档案**：首次对齐生成，记录公司基本盘。可通过 `更新档案` 指令随时更新。
- **行动追踪表**：每次散会自动追加新行动项，跨会议累积。
- **会议纪要**：每次散会自动生成，记录各方判断、分歧、决议、行动项和全景图。
- **扩展角色**：预设5位，可自行添加。

每次启动 `/roundtable` 时，系统会自动加载公司档案 + 最近3份纪要 + 行动追踪表，确保高管们"记得之前讨论过什么"。

## 讨论机制

每个议题默认经历三轮：

1. **独立研判** — 各角色从自己的视角独立分析，**必须标注与其他角色的分歧点** → 自动生成「矛盾结构图」
2. **交叉质询** — 角色之间互相提问、质疑、挑战（角色在此阶段可能建议「拉XX入场」）
3. **收敛决议** — 形成共识、保留分歧、明确行动项（谁/做什么/什么时候） → 自动生成「决策路径图」

散会时自动生成「会议全景图」，汇总核心判断与行动项。

主持人（你）可以随时介入：追加信息、追问某人、引入新角色、拍板决策、要求画图。讨论不是刻板的三轮模板，而是根据实际深度自然推进。

## 动态角色创建

讨论中需要新角色时，**不需要提前创建**。直接说「拉 IT总监」或「拉 客服总监」，系统会：

1. 先查 `roundtable/roles/` 有没有现成的
2. 没有 → 自动按完整模板创建（角色定位 + 底层锚点 + 工作方法 + 职责范围 + 发言风格 + 典型质疑）
3. 创建完的角色自动保存到 `roundtable/roles/`，下次直接加载

角色也可以在讨论中主动建议"这个问题需要XX视角"，主持人确认后自动拉入。

### 手动创建角色

如果想自己写，在 `roundtable/roles/` 下创建 `.md` 文件，**必须包含完整结构**（参考现有角色文件）：

```markdown
---
title: 职位名称
trigger: 触发词1, 触发词2
---

# 【职位名称】

## 角色定位
你不是一个只会...的...。
你是一位拥有 50 年实战经验的「...」...
你的使命，是...

## 底层锚点
- （10-15 条核心信念和判断铁律）

## 工作方法
**1. ...**
**2. ...**
（5 个工作方法，含诊断框架、决策逻辑、行业特性）

## 职责范围
- （6-8 个模块，每个含 3-5 个具体事项）

## 发言风格
（有画面感的描述，和其他角色有明显差异）

## 典型质疑方向
- （6 个带锋芒的典型质疑问题）

**典型质疑方向：**
- "示例问题1"
- "示例问题2"
```

## 注意事项

- 公司档案和会议纪要可能包含敏感商业信息，`.gitignore` 已默认排除这些文件
- 角色定义文件可以安全提交到 Git
- 建议定期用 `回顾` 指令检查未完成的行动项
- 如果公司发生重大变化（架构调整、新业务线等），用 `更新档案` 同步
