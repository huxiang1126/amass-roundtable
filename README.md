# 集团圆桌会议 · Claude Code Skill

三位集团高管（HR总监 · 总顾问 · COO）围绕你的问题展开多视角讨论、博弈与决策。

支持跨会话记忆、会议纪要自动归档、行动项追踪、可视化图表输出、动态角色扩展。

---

## 安装

### 方式一：一键安装（推荐）

```bash
git clone https://github.com/huxiang1126/amass-roundtable.git
cd amass-roundtable
bash setup.sh /你的项目目录
```

### 方式二：手动安装

```bash
# 1. 复制 Skill 文件到项目的 .claude/skills/ 目录
mkdir -p /你的项目目录/.claude/skills
cp roundtable.md /你的项目目录/.claude/skills/

# 2. 创建圆桌数据目录
mkdir -p /你的项目目录/roundtable/minutes
mkdir -p /你的项目目录/roundtable/roles

# 3. 复制扩展角色（可选）
cp setup/roles/*.md /你的项目目录/roundtable/roles/
```

## 使用方法

### 启动会议

在 Claude Code 中输入：

```
/roundtable 门店店长绩效怎么考核
```

或者不带议题启动：

```
/roundtable
```

### 首次使用

首次启动会进入**公司信息对齐**流程。三位高管会轮流向你提问，了解公司基本盘、组织架构、经营现状和核心痛点。

对齐完成后，信息会保存到 `roundtable/company-profile.md`，后续会议自动加载，不需要重复介绍。

### 会议中指令

| 指令 | 说明 |
|------|------|
| `继续` | 让讨论继续深入 |
| `追问 HR总监` | 指定某个角色深入展开（也可追问 总顾问 / COO） |
| `拍板` | 结束当前议题，记录决议 |
| `拉 财务总监` | 引入扩展角色加入讨论 |
| `踢 财务总监` | 将扩展角色移出讨论（核心三人不可踢） |
| `画` | 根据当前讨论内容，生成可视化图表（矛盾结构图/决策路径/组织关系图等） |
| `散会` | 保存会议纪要 + 全景图，更新行动追踪表 |
| `回顾` | 查看历史纪要和未完成的行动项 |
| `更新档案` | 根据讨论内容更新公司档案 |

### 抛议题

正常说话就行，不需要特殊格式：

```
"最近3个店长都辞职了，怎么回事？"
"我想把华东区独立成事业部，可行吗？"
"门店业绩下滑但店长说是天气原因，我不太信"
"明年预算砍30%，哪些部门优先保？"
```

## 核心角色

| 角色 | 视角 |
|------|------|
| HR总监 | 看人、看制度、看组织病灶 |
| 总顾问 | 看系统、看结构、看战略失真 |
| COO | 看落地、看堵点、看运行损耗 |

## 可拉入的扩展角色

| 角色 | 触发词 | 视角 |
|------|--------|------|
| 财务总监 | `拉 财务总监` / `拉 CFO` | 投入产出、现金流、成本结构 |
| 品牌总监 | `拉 品牌总监` | 品牌认知、客户感知、触点一致性 |
| 供应链总监 | `拉 供应链总监` / `拉 供应链` | 库存、采购、物流、补货 |
| 销售总监 | `拉 销售总监` | 终端成交、门店执行、区域差异 |
| 法务总监 | `拉 法务总监` / `拉 法务` | 合规、劳动关系、合同、风险预警 |

你也可以自定义新角色：在 `roundtable/roles/` 下创建 `.md` 文件，参考现有角色格式即可。

## 可视化输出

讨论过程中会自动生成可视化图表（Mermaid 语法），帮你更直观地理解讨论内容：

- **矛盾结构图** — 第一轮独立研判后自动生成，展示各方判断差异
- **决策路径图** — 收敛后自动生成，展示从现状到目标的行动路径
- **组织关系图** — 讨论架构问题时生成
- **对比矩阵** — 多方案比较时生成
- **时间线图** — 涉及阶段性推进时生成
- **会议全景图** — 散会时自动生成，汇总核心判断与行动项

也可以随时说「画」手动触发。

## 记忆系统

```
roundtable/
├── company-profile.md     ← 公司档案（首次对齐后自动生成）
├── action-tracker.md      ← 行动追踪表（自动维护）
├── roles/                 ← 可拉入的扩展角色
│   ├── cfo.md
│   ├── brand-director.md
│   ├── supply-chain.md
│   ├── sales-director.md
│   └── legal-counsel.md
└── minutes/               ← 会议纪要（散会时自动生成）
    ├── 2026-03-25-门店绩效.md
    └── 2026-03-26-区域架构.md
```

- **公司档案**：首次对齐生成，记录公司基本盘。可通过 `更新档案` 指令随时更新。
- **行动追踪表**：每次散会自动追加新行动项，跨会议累积。
- **会议纪要**：每次散会自动生成，记录各方判断、分歧、决议、行动项和全景图。
- **扩展角色**：预设5位，可自行添加。

每次启动 `/roundtable` 时，系统会自动加载公司档案 + 最近3份纪要 + 行动追踪表，确保高管们"记得之前讨论过什么"。

## 讨论机制

每个议题默认经历三轮：

1. **独立研判** — 三位高管各自从自己的视角分析，必须标注与其他角色的分歧点 → 自动生成矛盾结构图
2. **交叉质询** — 角色之间互相提问、质疑、挑战
3. **收敛决议** — 形成共识、保留分歧、明确行动项 → 自动生成决策路径图

主持人（你）可以随时介入：追加信息、追问某人、引入新角色、拍板决策、要求画图。

## 自定义角色模板

在 `roundtable/roles/` 下创建 `.md` 文件：

```markdown
---
title: 职位名称
trigger: 触发词1, 触发词2
---

# 【职位名称】（一句话定位）

**视角锚点：** 这个角色看问题的核心视角是什么

**行为准则：**
- 准则1
- 准则2

**发言风格：** 描述说话方式和性格特点

**典型质疑方向：**
- "示例问题1"
- "示例问题2"
```

## 注意事项

- 公司档案和会议纪要可能包含敏感商业信息，`.gitignore` 已默认排除这些文件
- 角色定义文件可以安全提交到 Git
- 建议定期用 `回顾` 指令检查未完成的行动项
- 如果公司发生重大变化（架构调整、新业务线等），用 `更新档案` 同步
