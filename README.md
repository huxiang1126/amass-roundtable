# 集团圆桌会议 · Claude Code Skill

三位集团高管（HR总监 · 总顾问 · COO）围绕你的问题展开多视角讨论、博弈与决策。

支持跨会话记忆、会议纪要自动归档、行动项追踪、可视化图表输出、动态角色扩展。

---

## 安装

### 前提条件

- 已安装 [Claude Code 桌面应用](https://claude.ai/download)
- 已安装 Git（Mac 自带）

### 第一步：下载源码

打开「终端」应用，粘贴以下命令：

> Mac 终端位置：「启动台」→「其他」→「终端」

```bash
git clone https://github.com/huxiang1126/amass-roundtable.git
```

这会在你当前目录下创建一个 `amass-roundtable` 文件夹。

### 第二步：选择安装位置并执行安装

你需要准备一个文件夹作为「圆桌会议」的工作目录。可以是任意位置，比如：

```bash
# 示例 1：安装到桌面
bash amass-roundtable/setup.sh ~/Desktop/圆桌会议

# 示例 2：安装到文档目录
bash amass-roundtable/setup.sh ~/Documents/圆桌会议

# 示例 3：安装到已有的项目目录
bash amass-roundtable/setup.sh /你的项目路径
```

文件夹不存在会自动创建。看到 `安装完成！` 就装好了。

### 第三步：在 Claude Code 中打开

1. 打开 **Claude Code 桌面应用**
2. 新建对话，点击左下角的项目目录选择器
3. 选择第二步中你安装到的那个文件夹（比如 `圆桌会议`）
4. 在对话框输入 `/roundtable`，出现命令提示说明安装成功

> **注意**：Claude Code 只能识别当前项目目录下安装的 Skill。如果切换了项目目录，`/roundtable` 不会出现。确保你选的是安装了 Skill 的那个文件夹。

### 安装后的目录结构

```
你选择的文件夹/
├── .claude/
│   └── skills/
│       └── roundtable/
│           └── SKILL.md          ← Skill 核心文件
└── roundtable/
    ├── roles/                    ← 预装的扩展角色
    │   ├── cfo.md
    │   ├── brand-director.md
    │   ├── supply-chain.md
    │   ├── sales-director.md
    │   └── legal-counsel.md
    ├── minutes/                  ← 会议纪要（自动生成）
    └── .gitignore
```

## 使用方法

### 启动会议

在 Claude Code 对话框中输入：

```
/roundtable 门店店长绩效怎么考核
```

或者不带议题直接启动：

```
/roundtable
```

### 首次使用

首次启动会进入**公司信息对齐**流程。三位高管会轮流向你提问，了解公司基本盘、组织架构、经营现状和核心痛点。

对齐完成后，信息会保存到 `roundtable/company-profile.md`，后续会议自动加载，不需要重复介绍。

> 对齐过程中直接打字回答即可，不需要每次都加 `/roundtable` 前缀。只有启动会议时需要输入 `/roundtable`。

### 会议中指令

| 你说 | 效果 |
|------|------|
| 直接说话 | 抛出议题或补充信息，高管们会接着讨论 |
| `继续` | 让讨论继续深入 |
| `追问 HR总监` | 指定某个角色深入展开（也可追问 总顾问 / COO） |
| `拍板` | 结束当前议题，记录决议 |
| `拉 财务总监` | 引入新角色参与讨论（角色库没有的会自动创建） |
| `踢 财务总监` | 将扩展角色移出讨论（核心三人不可踢） |
| `画` | 生成当前讨论的可视化图表 |
| `散会` | 保存会议纪要 + 全景图，更新行动追踪表 |
| `回顾` | 查看历史纪要和未完成的行动项 |
| `更新档案` | 根据讨论内容更新公司档案 |

### 抛议题示例

正常说话就行，不需要特殊格式：

```
"最近3个店长都辞职了，怎么回事？"
"我想把华东区独立成事业部，可行吗？"
"门店业绩下滑但店长说是天气原因，我不太信"
"明年预算砍30%，哪些部门优先保？"
```

## 核心角色

| 角色 | 视角 | 风格 |
|------|------|------|
| HR总监 | 看人、看制度、看组织病灶 | 表面是人的问题，往深里都是制度的问题 |
| 总顾问 | 看系统、看结构、看战略失真 | 局部最优不代表全局最优 |
| COO | 看落地、看堵点、看运行损耗 | 战略不落到动作就是空谈 |

三位高管会从各自角度分析，**必须指出彼此的分歧**，不会一团和气。

## 扩展角色

### 预装角色（开箱即用）

| 角色 | 触发词 | 视角 |
|------|--------|------|
| 财务总监 | `拉 财务总监` / `拉 CFO` | 投入产出、现金流、成本结构 |
| 品牌总监 | `拉 品牌总监` | 品牌认知、客户感知、触点一致性 |
| 供应链总监 | `拉 供应链总监` / `拉 供应链` | 库存、采购、物流、补货 |
| 销售总监 | `拉 销售总监` | 终端成交、门店执行、区域差异 |
| 法务总监 | `拉 法务总监` / `拉 法务` | 合规、劳动关系、合同、风险预警 |

### 没有的角色？直接拉

不需要提前创建。直接说 `拉 IT总监` 或 `拉 客服总监`，系统会自动创建完整的角色定义并保存，下次会议直接加载。

讨论中高管也会主动建议「这个问题需要XX视角」，你确认后自动拉入。

## 讨论机制

每个议题默认经历三轮：

| 轮次 | 内容 | 自动输出 |
|------|------|----------|
| 第1轮 · 独立研判 | 各角色独立分析，标注与其他角色的分歧 | 矛盾结构图 |
| 第2轮 · 交叉质询 | 互相提问、质疑、挑战 | — |
| 第3轮 · 收敛决议 | 形成共识，列出行动项（谁/做什么/什么时候） | 决策路径图 |

散会时自动生成「会议全景图」，汇总核心判断与行动项。

你可以随时介入：追加信息、追问某人、引入新角色、拍板决策、要求画图。讨论根据实际深度自然推进，不是死板的三轮模板。

---

## 可视化输出

讨论过程中会自动生成图表（Mermaid 语法）：

- **矛盾结构图** — 展示各方判断差异
- **决策路径图** — 从现状到目标的行动路径
- **组织关系图** — 讨论架构问题时生成
- **对比矩阵** — 多方案比较时生成
- **时间线图** — 涉及阶段性推进时生成
- **会议全景图** — 散会时汇总全部

也可以随时说「画」手动触发。

## 记忆系统

| 文件 | 说明 | 何时生成 |
|------|------|----------|
| `roundtable/company-profile.md` | 公司档案 | 首次对齐后自动生成 |
| `roundtable/action-tracker.md` | 行动追踪表 | 首次散会后自动生成，跨会议累积 |
| `roundtable/minutes/日期-主题.md` | 会议纪要 | 每次散会时自动保存 |
| `roundtable/roles/*.md` | 角色定义 | 预装5位 + 动态创建自动保存 |

每次启动 `/roundtable` 时，系统自动加载公司档案 + 最近3份纪要 + 行动追踪表，高管们「记得」之前讨论过什么。

---

## 注意事项

- 公司档案和会议纪要可能包含敏感商业信息，`.gitignore` 已默认排除这些文件
- 角色定义文件可以安全提交到 Git
- 建议定期用「回顾」指令检查未完成的行动项
- 如果公司发生重大变化（架构调整、新业务线等），用「更新档案」同步
